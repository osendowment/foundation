# Comparison to existing solutions

Current funding solutions are not enough:

|                                                                                                                            | Pays maintainers   | Stable funding     | Decentralised authority | Funds globally critical projects |
| -                                                                                                                          | -                  | -                  | -                       | -                                |
| **Tech corporations**                                                                                                      | :white_check_mark: | :x: <sup>1</sup>   | :x: <sup>2</sup>        | :x: <sup>2</sup>                 |
| **Open source foundations**<br>eg [Linux Foundation][linux-foundation],<br>[Apache Software Foundation][apache-foundation] | :x: <sup>3</sup>   | :x: <sup>1</sup>   | :question:              | :x: <sup>4</sup>                 |
| **[thanks.dev][thanksd]**                                                                                                  | :white_check_mark: | :x: <sup>1</sup>   | :x:                     | :white_check_mark:               |
| **[GitHub Sponsors][gh-sponsors]**, **[Open Collective][opencollective]** and **[Polar][polar]**                           | :white_check_mark: | :x: <sup>5</sup>   | :white_check_mark:      | :x: <sup>5</sup>                 |
| **Government programmes**<br>eg [Sovereign Tech Agency][sovtech]                                                           | :white_check_mark: | :white_check_mark: | :question:              | :x: <sup>6</sup>                 |
| **Open Source Endowment**                                                                                                  | :white_check_mark: | :white_check_mark: | :white_check_mark:      | :white_check_mark:               |

<sup>1</sup> Depends on fickle corporate budgets<br>
<sup>2</sup> Prioritises corporate interests<br>
<sup>3</sup> Most foundations currently do not pay maintainers as a rule<br>
<sup>4</sup> Prioritises specific ecosystems<br>
<sup>5</sup> Projects popular with end-users receive funding<br>
<sup>6</sup> Focuses on national interests (though [Sovereign Tech Agency][sovtech] does have a global focus)

Nobody had built an open source software endowment before OSE, because:

* **The endowment model is capital-intensive and requires a 20+ year planning horizon**, because only ~5% of donations
  are spent every year. Most charities focus on shorter-term results.
* **The endowment model is best suited to very long-term goals** such as the sustainability of open source software
  generally, but a lot of existing OSS funding is project-specific, targeting operating systems, language ecosystems, or
  specific developers.
* **Building an endowment for OSS requires a rare intersection of multiple areas of expertise** — software development,
  up-to-date knowledge on open source sustainability, fundraising, endowments, startup scaling and so on. Most
  philanthropists are not familiar with open source values and their challenges — this problem did not exist 25 years
  ago.
